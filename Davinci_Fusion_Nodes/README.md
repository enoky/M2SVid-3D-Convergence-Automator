# M2SVid 3D Convergence Automator — DaVinci Resolve Fusion Macro

A pre-built Fusion node graph (`.setting` file) that applies automated convergence control to a full side-by-side stereoscopic 3D clip inside DaVinci Resolve. It reads the grayscale convergence control video produced by the M2SVid 3D Convergence Automator and uses it to dynamically shift and scale both eye views on every frame — no manual keyframing required.

## Node Graph Overview

```
Full_SBS_Input (MediaIn)           Convergence_Input (MediaIn)
       │                                     │
   Splitter                               Probe (reads luminance → 0.00–0.04)
    ┌──┴──┐                                  │
  Left   Right                         Control_Panel
    │      │                          (Convergence slider)
 GlobalAlign ──────────────────────────────── ↑
    ┌──┴──┐                  expression: Translation.X = -Convergence
    │      │
Transform1  Transform2       expression: Size = Convergence + 1.01
    │      │
  Combiner (SBS → single frame)
       │
   Anaglyph (preview / passthrough)
       │
   MediaOut
```

**How it works:** A `Probe` node samples the center pixel of the convergence control video each frame, mapping its brightness to a value between 0 and 0.04. That value drives the `GlobalAlign` horizontal shift (pushing the left/right views apart) and both `Transform` nodes' scale (zooming in slightly to hide the shifted edges). The net effect is a per-frame convergence adjustment that tracks the most salient depth in the scene.

## Installation

### Method 1 — Install as a Fusion Template (Recommended)

This makes the macro available from the **Templates** toolbar inside Fusion for all future projects.

1. **Locate your Fusion Templates folder:**

   | OS | Path |
   |---|---|
   | **Windows** | `C:\ProgramData\Blackmagic Design\DaVinci Resolve\Fusion\Templates\Edit\` |
   | **macOS** | `/Library/Application Support/Blackmagic Design/DaVinci Resolve/Fusion/Templates/Edit/` |
   | **Linux** | `/var/BlackmagicDesign/DaVinci Resolve/Fusion/Templates/Edit/` |

   > [!TIP]
   > You can also use the **per-user** templates path:
   > - **Windows:** `%APPDATA%\Blackmagic Design\DaVinci Resolve\Support\Fusion\Templates\Edit\`
   > - **macOS:** `~/Library/Application Support/Blackmagic Design/DaVinci Resolve/Fusion/Templates/Edit/`

2. **Create a subfolder** (optional, for organization):
   ```
   Templates\Edit\M2SVid\
   ```

3. **Copy** `M2SVid-3D-Convergence-Automator.setting` into that folder.

4. **Restart DaVinci Resolve** (or close and reopen the Fusion page).

5. The macro will now appear in the **Effects Library → Templates → Edit → M2SVid** panel on the Fusion page.

---

### Method 2 — Drag & Drop into a Fusion Composition

If you just want to use it once without installing:

1. Open your project in DaVinci Resolve.
2. Switch to the **Fusion** page (or open a clip in Fusion from the Edit page).
3. Open a **File Explorer / Finder** window and navigate to the folder containing `M2SVid-3D-Convergence-Automator.setting`.
4. **Drag the `.setting` file** directly onto the Fusion **node editor canvas**.
5. The entire node graph will be pasted into your composition.

---

### Method 3 — Paste via the Fusion Menu

1. On the **Fusion** page, right-click the node editor canvas.
2. Select **Settings → Load Settings…** (or go to **File → Import → Fusion Settings…** in some versions).
3. Browse to `M2SVid-3D-Convergence-Automator.setting` and open it.
4. The nodes will be inserted into the current composition.

## Usage

### Prerequisites

Before using this macro you need:

1. A **full side-by-side (SBS) 3D video** — the stereoscopic footage you want to apply convergence to.
2. A **convergence control video** (`M2SVid_Convergence_Control.mp4`) — generated by the M2SVid 3D Convergence Automator tool (see the [main README](../README.md)).

> [!IMPORTANT]
> Both videos must have the **same frame count and frame rate**. The convergence control video is a 1:1 frame-for-frame map of the SBS footage.

### Step-by-Step

1. **Import both videos** into the DaVinci Resolve **Media Pool**:
   - Your full SBS 3D video
   - The `M2SVid_Convergence_Control.mp4` file

2. **Add the SBS video** to the timeline on the **Edit** page.

3. **Open the clip in Fusion** — select the clip on the timeline and click the **Fusion** tab (or press `Ctrl+Shift+F` / `Cmd+Shift+F`).

4. **Load the macro** using any of the three installation methods above.

5. **Connect the inputs:**

   | Node | Connect To |
   |---|---|
   | **`Full_SBS_Input`** | Your full side-by-side 3D `MediaIn` node (the clip on the timeline, or drag it from the Media Pool) |
   | **`Convergence_Input`** | Drag `M2SVid_Convergence_Control.mp4` from the Media Pool onto the Fusion canvas to create a new `MediaIn`, then connect it to this input |

   > [!NOTE]
   > If you loaded the macro into an existing composition that already has a `MediaIn` for the timeline clip, simply connect that existing `MediaIn`'s output to the `Full_SBS_Input` node's input.

6. **Connect the output** — wire the `MediaOut1_1` node to your composition's `MediaOut` (or replace the default `MediaOut`).

7. **Play back** the timeline. The convergence will adjust automatically on every frame.

### Previewing

- The `Anaglyph` node is set to **pass-through** by default. To preview the 3D effect as a red/cyan anaglyph:
  1. Select the `Anaglyph1` node.
  2. Uncheck **Pass Through** in the Inspector.
  3. Choose your preferred stereo display method (Anaglyph, Difference, etc.).

### Adjusting Convergence Strength

- Select the **`Control_Panel`** node (highlighted in orange).
- In the Inspector, you'll find the **Convergence** slider (range: `0.00` – `0.04`).
- This value is driven automatically by the `Probe` node reading the convergence video, but you can:
  - **Override** it by disconnecting the Probe and keyframing the slider manually.
  - **Scale** the effect by modifying the expressions on `Transform1` / `Transform2` (e.g., change `Convergence + 1.01` to `Convergence * 0.5 + 1.01` to halve the zoom).

## Troubleshooting

| Issue | Solution |
|---|---|
| Nodes don't appear after import | Restart DaVinci Resolve. Ensure the `.setting` file is in the correct Templates directory and not inside a nested subfolder beyond what's expected. |
| Black output or no convergence effect | Verify both `MediaIn` nodes are connected and that the convergence video has the same frame count / frame rate as the SBS clip. |
| Edge clipping after convergence shift | The `Transform` nodes zoom in slightly (≈1%–5%) to compensate. If you see black edges, increase the `Size` expression constant (e.g., `Convergence + 1.02`). |
| `Probe` reads 0 on every frame | Make sure `Convergence_Input` is connected to the correct convergence grayscale video, not an empty or wrong clip. The Probe reads the **alpha channel** (`Channel = 4`) — if your video has no alpha, change the Probe's Channel to **Luminance** in the Inspector. |
