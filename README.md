# M2SVid 3D Convergence Automator

A desktop GUI tool that automatically generates **convergence control videos** for stereoscopic 3D movies created with [M2SVid](https://github.com/Archit01/M2SVID-gui). It uses a trained neural network to analyze synchronized RGB + depth video pairs and produce a grayscale control track that drives the Convergence parameter in DaVinci Resolve — eliminating the need for manual per-frame keyframing.

---

## How It Works

The tool takes two synchronized inputs for each clip:

| Input | Description |
|---|---|
| **RGB video** | The original color footage |
| **Depth video** | A matching depth map video (e.g., generated by DepthCrafter) |

A **U²-Net**-based saliency model (`ConvergenceEstimator`) identifies the most visually important regions in each frame and cross-references their depth values to predict an optimal convergence point. The raw predictions are then stabilized through a multi-stage smoothing pipeline:

1. **EMA MinMax Scaling** — normalizes depth values with an exponential moving average over a sliding window to prevent sudden range shifts.
2. **Per-frame EMA Smoothing** — blends each prediction with the previous value for temporal coherence.
3. **Post-process Temporal Convolution** — applies a centered box filter across the entire clip to remove residual flicker.

The final output is streamed directly into a single, continuous grayscale `.mp4` file (`M2SVid_Convergence_Control.mp4`), where each frame's brightness (0–255) encodes the convergence value for that moment in the film. By maintaining a single video writer stream, the process avoids heavy temporary disk I/O.

---

## Features

- **Fully GPU-accelerated pipeline** — model inference, buffer normalizations, and even frame color-space conversions execute natively on PyTorch CUDA arrays to completely avoid CPU bottlenecks.
- **Batch processing** — processes an entire folder of video clips in sequence.
- **Single-stream rendering** — chunks are appended sequentially into one continuous timeline-ready control video, completely bypassing the need for temporary files and external concatenators.
- **Tunable parameters** — convergence ratio, EMA alpha, scaler decay, buffer size, and temporal window are all exposed in the GUI.
- **Singleton model loading** — the neural network is loaded once and reused across all clips.

## Installation

```bash
# Clone the repository
git clone https://github.com/enoky/M2SVid-3D-Convergence-Automator.git
cd M2SVid-3D-Convergence-Automator

# Create and activate a virtual environment
python -m venv venv
venv\Scripts\activate        # Windows
# source venv/bin/activate   # Linux / macOS

# Install dependencies (pulls PyTorch with CUDA 12.8 support)
pip install -r requirements.txt
```

> [!NOTE]
> The pre-trained model weights (`iw3_sod_v1_20260125.pth`, ~4.6 MB) are included in the repository. No separate download is required.

## Usage

### Launch the GUI

**Windows (double-click):**
```
RUN_Convergence_Automator.bat
```

**Manual:**
```bash
python app.py
```

### Workflow

1. **Input (RGB) Folder** — select the folder containing your original color video clips (`.mp4`, `.mov`, `.avi`).
2. **Input (Depth) Folder** — select the folder containing the matching depth videos. Each depth file must be named `<clip_name>_depth.<ext>` (e.g., `scene01.mp4` → `scene01_depth.mp4`).
3. **Output Folder** — choose where the convergence control video will be saved.
4. **Adjust parameters** (optional) — tune the smoothing and convergence settings below.
5. Click **Start Processing**.

The final output, `M2SVid_Convergence_Control.mp4`, can then be imported into DaVinci Resolve as a control track.

## Parameters

| Parameter | Default | Range | Description |
|---|---|---|---|
| **Convergence Ratio** | `0.6` | 0.0 – 1.0 | Controls where the convergence plane sits relative to the salient object's depth range. Lower values push the plane closer to the camera; higher values push it deeper. |
| **EMA Alpha** | `0.2` | 0.01 – 1.0 | Frame-to-frame smoothing factor. Lower values produce smoother (more sluggish) transitions; higher values track predictions more closely. |
| **Scaler Decay** | `0.9` | 0.0 – 1.0 | How quickly the MinMax normalization adapts to new depth ranges. Higher values create a more stable baseline; lower values react faster. |
| **Scaler Buffer** | `60` | ≥ 1 | Number of frames used in the MinMax sliding window. Larger buffers create a wider temporal context for normalization. |
| **Temporal Window** | `31` | ≥ 1 (odd) | Width of the post-process box filter (in frames). Larger windows produce a smoother final curve. Even values are automatically incremented to the next odd number. |

## Architecture Overview

```
RGB Frame ──┐
             ├──► Preprocess (resize 192×192, concat 6-ch) ──► U²-Net P ──► Saliency Map ─┐
Depth Frame ─┘                                                                             │
     │                                                                                     │
     └──► EMAMinMaxScaler (normalize) ──────────────────────────► Depth Map ────────────────┤
                                                                                           │
                                                              calculate_robust_depth() ◄───┘
                                                                       │
                                                                 raw prediction
                                                                       │
                                                              EMA Smoothing (α)
                                                                       │
                                                          Temporal Box Filter (window)
                                                                       │
                                                              Grayscale Frame
                                                             (brightness = value)
```
